Designing & Creating A
Spelling Game


(Spell Well)


Roles: UI designer and Web Designer/Developer

Tools: Adobe XD, Dreamweaver, Wordpress

Skills: Wireframing, Usability Testing

Time: Four Weeks





Problem:


Need a place to host the game to its users and where users can learn how to play and learn more about the game.

Solution (Goal):


The solution is to create a website that can display the information about the game, a place for the game to run and a feedback area (where users can give feedback).




About 'Spell Well':

This game helps those who are struggling with keeping up with school work and or are burnt out from studying and need for a new/unique way of memorizing words. This idea uses a similar feature of “quizlet” where the users are able to input information and descriptions of what they need to memorize. In the site, users can input words that they need practice (or help with) along with their definitions, then during gameplay, players will try to guess words based on the definition that shows on screen.


Plan:


The aim of the project is to go through the design process and create low and high fidelity wireframes, and then develop the game using html, css and js. The content of the webpage includes the game page which has the actual game where users can play. In the game page, users will be able to submit their words and definitions to revise. Or choose from a preset list of dictionaries. The project intends to have a feedback page where it will contain a form page, where users can interact with and submit messages. The website features different types of multimedia such as images, text, and links to social media accounts. On the right, is a list of the high and low priority functions for the gameplay.



Information Architecture (Sitemap):



The site map has three main pages: The welcome page which includes the Spell Well's landing page and the about page. The second page is the game itself, and third is the feedback page.

Welcome Page:
The welcome page provides instructions and information about the game.

Game Page:
The game page has two options: input definitions and word, or random word generator. After one of the options are selected the spelling game will start.

Feedback Page:
This page has a form that users can fill to provide feedback.

In the footer, the users will find the social media handles of the game.

Low - Fidelity Wireframe:


This wireframe has the navigation stuck to the beginning of the site. Both of the low-fidelity wireframes represent the homepage. This wireframe has a big header with a button on the right side. And on the left side beside the button, represents an image. Next, is a section for the game and has a button called 'play'.



High - Fidelity Wireframe:


This wireframe has a navigation stuck to the right side of the site. The navigation menu takes the form of a ‘pill’ shaped border. The homepage introduces the user to the game site and has the option to play the game or learn more. The footer is simple and has the slogan of the game on the right and the social media icons on the right.



Viusal Identity:


For the game, I decided to use the colors pink (#cca0bb), dark pink (#c17b9f), and dark purple (#2f073b). The font decided was displayfair as it allows easy readability. These colors and the font are used throughout the whole website to create consistency.




Usability Testing:


The usability evaluation helped find the small details that were missed. These include:

- Having the title 'Spell Well' next to the logo (so users know which website they are on at all times)
- "Remove the 'about' page and add content on the about page to the home page" (originally the 'about' section was its own page)
- "Change the color of the global menu to have better contrast" (white and light pink don't go. Hence, it was changed to black and light pink)
- Clarify how the scores are calculated in the instructions section.
- "Add an explanation on feedback form"
- "Be able to delete dictionaries"
- "Prevent users from having duplicate dictionaries"
